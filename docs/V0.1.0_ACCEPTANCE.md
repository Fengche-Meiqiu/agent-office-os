# V0.1.0 Acceptance Criteria

V0.1.0 is the first deployable foundation release for Agent Office OS.

## User Goal

A user can open Agent Office OS and complete the core Hermes-backed workflow:

```text
sync Hermes agents
-> review the talent marketplace
-> hire one agent into the office
-> chat with that agent
-> create a /task request
-> monitor the task
-> view the result in Outputs
```

## Required User-Facing Scope

1. Talent marketplace is usable.
   - The marketplace can sync agents from Hermes.
   - The user can manually refresh the talent library.
   - Agent cards show identity, role/title, platform, status, description, skills, and tools when Hermes provides them.
   - Missing Hermes fields must be visible as missing or pending, not silently replaced with fake data.

2. Office is usable.
   - The user can hire an agent from the marketplace.
   - Hired agents appear in the office.
   - The user can open an agent profile.

3. Single-agent chat is usable.
   - Default mode is chat.
   - User messages are sent to Hermes through the backend adapter.
   - Hermes replies are displayed in the chat UI.
   - Failures return a clear error instead of hanging silently.

4. Task mode is usable.
   - Messages beginning with `/task` create backend tasks.
   - Tasks are dispatched to Hermes.
   - Task status is visible in the task center.
   - Completed task results are stored in Outputs when Hermes returns them.

5. Outputs are usable.
   - Task outputs are listed.
   - Text/markdown outputs can be opened and read.

6. Meeting mode remains basic.
   - Basic meeting create/start/ask-all/ask-agent/finish behavior may remain.
   - Automatic endless discussion, pause/resume, and Nuwa-generated summaries are deferred to V0.2.0.

7. Organization structure is removed.
   - No sidebar entry.
   - No frontend route.
   - No backend organization API route.

## Engineering Gates

These commands must pass before a V0.1.0 release is cut:

```bash
npm run check
npm run build
python -m compileall backend/app
python -m pytest backend/tests
```

## Deployment Gate

The release package must deploy through the established chain:

```text
GitHub Release
-> Silicon Valley mirror
-> Tencent Cloud Hermes watcher
-> deployment report
```

The deployment report must show:

- package downloaded
- sha256 verified
- deploy script succeeded
- check script succeeded
- deployed version recorded
