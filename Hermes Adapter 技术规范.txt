# Hermes Adapter 技术规范

## Hermes Adapter Technical Specification

Version: V1.0

---

# 1. 文档目标

定义Hermes与Office OS之间的标准接入协议。

Hermes作为Agent Runtime。

Office OS作为Agent Management System。

两者通过Agent Gateway进行通信。

---

# 2. 整体架构

Office UI

↓

Office Backend

↓

Agent Gateway

↓

Hermes Adapter

↓

Hermes

---

# 3. Adapter职责

Hermes Adapter负责：

1. Agent发现
2. Agent信息同步
3. Agent聊天
4. Agent任务派发
5. Agent状态同步
6. Webhook接收
7. 数据格式转换

---

# 4. Agent发现

## 目标

同步Hermes中的Agent。

---

接口：

discover_agents()

返回：

[
{
"id":"hermes_agent_001",
"name":"市场分析师",
"description":"负责市场研究",
"status":"online"
}
]

---

同步周期：

5分钟

支持手动刷新

---

# 5. Agent Profile

接口：

get_profile(agent_id)

返回：

{
"id":"hermes_agent_001",

"name":"市场分析师",

"role":"Market Analyst",

"description":"负责行业研究",

"platform":"Hermes"
}

# 8. Memory同步

接口：

返回：

{
"summary":"负责新能源行业研究"
}

不支持编辑。

# 10. Task

接口：

{
"title":"分析新能源汽车行业",

}

返回：

{
"task_id":"task_123"
}

---

# 11. Task Query

接口：

get_task(task_id)

返回：

{
"status":"running"
}

{
"status":"completed",

"result":"..."
}

# 13. Webhook

Hermes主动通知Office。

POST

/api/webhook/hermes

---

支持事件：

task_created

task_started

task_completed

task_failed

agent_online

agent_offline

---

示例：

{
"event":"task_completed",

"task_id":"123",

"agent_id":"001",

"status":"completed",

"result":"..."
}

---
